# 配置 iptables 和 Fail2Ban 增强安全防护

# iptables:防火墙框架

iptables 是一个允许灵活配置的安全策略的防火墙框架（防火墙） 原则是设置一些出入的规则，尽量筛选掉一些恶意的恶意的访问和流量；

升级和更新 ubantu：

sudo apt-get update && sudo apt-get upgrade

清空现有的 iptables 规则：

sudo iptables -F

添加规则：

sudo vi /etc/iptables.up.rules

	*filter
	-A INPUT -m state --state ESTABLISHED,RELATED -j ACCEPT    #允许所有建立起来的链接
	-A OUTPUT -j ACCEPT    #允许所有出去的流量
	-A INPUT -p tcp --dport 443 -j ACCEPT    #允许https协议下的请求连接
	-A INPUT -p tcp --dport 80 -j ACCEPT   #允许80端口流量进出
	-A INPUT -p tcp -m state --state NEW --dport 49999 -j  ACCEPT    #允许terry用户通过ssh登录方式进入到服务器的这个端口
	-A INPUT -p icmp -m icmp --icmp-type 8 -j ACCEPT    #允许从外网ping到这台服务器上（方便测试服务器有没有停机等等）
	-A INPUT -m limit --limit 5/min -j LOG --log-prefix "iptables denied:" --log-level 7    #记录被拒绝的请求
	#对恶意访问（可疑或密集请求）网站的ip加以拦截
	-A INPUT -p tcp --dport 80 -i eth0 -m state --state NEW -m recent --set
	-A INPUT -p tcp --dport 80 -i eth0 -m state --state NEW -m recent --update --seconds 60 --hitcount 150 -j DROP           #若某IP对80端口60秒内发出超过150次请求，则认为是敏感访问并拦截
	#拒绝其他的进服务器的流量
	-A INPUT -j REJECT
	-A FORWARD -j REJECT
	COMMIT

告诉 iptables 配置文件在哪里：

sudo iptables-restore < /etc/iptables.up.rules

没报错则防火墙通过了；

查看防火墙有没有成功启动：

sudo ufw status

由于没有激活，则要激活 ufw：

sudo ufw enable

输入 y 激活；

设置开机自启防火墙：

通过创建脚本：sudo vi /etc/network/if-up.d/iptables

写入脚本：

#！/bin/sh
iptables-restore /etc/iptables.up.rules

sudo chmod +x /etc/network/if-up.d/iptables

完成 iptables；

# Fail2Ban:防御性动作库

通过监控系统的日志文件，更具检测到的任何可疑行为触发不同的防御动作（如对可疑行为执行 ip 锁定）

安装：

sudo apt-get install fail2ban

安装后打开配置文件：

sudo vi /etc/fail2ban/jail.conf

保存；

查看 fail2ban 有没有运行：

sudo service fail2ban status

停掉：sudo service fail2ban stop

开始：sudo service fail2ban start

综上，比起裸的服务器安全级别已经高很多了；
