# 配置 Nginx 反向代理 Nodejs 端口

解决 WEB 服务可以通过 80 端口被外网访问到，通过 80 端口无法启动原因是用户下不具备 root 的运行权限，不能监听 0-1024 下的任何端口；

引入 Nginx 用 root 级别的权限来启动对 80 端口的监听，同时把来自 80 端口的流量分配给 node 服务的另外一个端口，实现服务的代理：

  • 若服务器只需放一个网站程序的话，网站程序监听 80 端口即可；
  • 若服务器有多个应用，借助 Nginx 可以实现端口的代理，还可以实现负载均衡，让它判断是来自那个域名或 IP 的访问，根据规则将请求原封不动的转发给特定端口或特定的机器（如将 80 端口的请求都转到 node 启动的 8081 端口来处理）

安装 Nginx：

停止阿帕奇：sudo service apache2 stop ; sudo service apache stop

删除阿帕奇：update-rc.d -f apache2 remove

移除阿帕奇：sudo apt-get remove apache2

更新包：sudo apt-get update

安装 Nginx：sudo apt-get install nginx

检查版本：nginx -v

进入 nginx 目录下的 conf.d 文件夹：cd /etc/nginx/conf.d

新建配置文件：sudo vi webone-com-8082.conf //命名规则：域名（webone.com）-其背后的端口（8082）

	#为考虑多个项目对应的多个服务的负载均衡的场景，
	#通过upstream+应用名字设定简单策略；
	upstream webone {    #定义webone
		server 127.0.0.1:8082;
	}
	server {    #定义server
		listen 80;
		server_name 60.205.190.154;
		location / {
			proxy_set_header X-Real-IP $remote_addr;
			proxy_set_header X-Forward-For $proxy_add_x_forwarded_for;
			proxy_set_header Host $http_host;
			proxy_set_header X-Nginx-Proxy true;
			proxy_pass http://webone;
			proxy_redirect off;
		}
	}
