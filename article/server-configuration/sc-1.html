# 远程登陆服务器

# 第一次 ssh 远程登陆服务器（可通过 wsl 使用）

命令行下登录方式：

通过ssh  用户名（阿里云默认root）@主机公网IP + 回车

提示把你的 key 指纹加入到本地信任列表里面；

输入 yes 后提示输入密码；

输入成功后回车： 登进去后简单的欢迎信息；

数据放在系统盘重装系统后数据会丢失，挂载在数据盘的话会提高安全性；

查看方式：fdisk -l

Ctrl + L 清空面板；

df -h 查看硬盘使用情况：

Ctrl + D 退出环境；

# 配置 root 及应用账号权限

增加用户：

adduser  +  用户名

对刚设置的用户授权：

gpasswd -a 该用户名 sudo

（使该用户以 sudo 角色调用系统命令，具备部分超级管理员的能力，但是必须输入密码）升权

通过 sudo visudo 出现一大段内容的配置文件：

terry ALL=(ALL:ALL) ALL 给该用户和 root 同样高的权限；

按 Ctrl + X ：

保存：按 Shift + Y 回车；

测试：

若无法连接，在 root 的登录下执行：server ssh restart 重启 ssh

# 配置本地无密码 SSH 登录

原理：

查看本地电脑之前是否配置过 ssh 私钥：

  • ls -a 查看有没有.ssh 文件夹，若有就 cd .ssh 并 ls 若为空就是还没有配置过，不为空则配置过；
  • 若没有.ssh 文件夹就要新建 mkdir .ssh；
  • 再生成公钥和私钥：

    ssh-keygen -t rsa -b 4096 -C "muyijuexian@163.com" 回车

在次查看：

查看内容：

cat id_rsa     查看私钥

cat id_rsa.pub     查看公钥

生成公钥私钥后，开启 ssh 代理：

在 ssh 文件夹中执行：eval "$(ssh-agent -s)"

把 ssh 的 key 加入到代理中：

ssh-add ~/.ssh/id_rsa

新开一个 TAB 连服务器：

没有，则服务器上生成公私钥：

在 ssh 文件下创建授权文件：

vi authorized_keys(授权文件中的钥匙是可信的，只对当前登录用户生效)

按Shift + ：

输入wq!  回车；

再查看：

有了授权文件后，需要把本地电脑的公钥放到该文件中去，当电脑登陆服务器时，服务器会验证它提交的私钥与之前拷贝过来的私钥匹配之后，服务器才会向他敞开大门；

切换到本地命令行下：

  • cat id_rsa.pub打印公钥内容；

  • 光标复制整段内容；
  • 切换到服务器命令行窗口下通过 vi 打开 authorized_keys：
  • vi authorized_keys
  • 按下 i 进入编辑模式；
  • 粘贴内容；
  • 按下 Esc
  • 再按 Shift + ：
  • 输入 wq！ 回车；存入内容；
  • 给 authorized_keys 文件授权：chmod 600 authorized_keys
  • 重启 ssh 服务：sudo service ssh restart
